Client sends AI-extracted remittance data: customer name, payment reference, date, amount, and invoice lines.
Pydantic Validation models.py ▼
Validates and cleans all fields. Empty strings become null. Strings are stripped. Unknown fields silently ignored.
Fetch Oracle Reports (Parallel, Cached) ORACLE ▼
Two threads fetch receipt + invoice reports simultaneously. Data is cached for 5 minutes. SOAP 1.2 call, base64 CSV response.
Receipt Matching: SUB 1 (Unapplied) → SUB 2 (Applied) MATCH ▼
SUB 1 (Cached Unapplied): No SOAP params sent — Oracle returns only unapplied (UNAPP/UNID) receipts. Run cascading match A1→A7 or B1→B4.

SUB 2 (Filtered Applied): Only triggered if SUB 1 finds no match. Sends a single SOAP parameter at a time (Oracle returns empty with multiple). Tries sequentially: P_CUSTOMER_NAME → P_RECEIPT_NUMBER → P_RECEIPT_AMOUNT → P_RECEIPT_DATE until one returns data. Run same matching on results.
Invoice Matching: SUB 1 (Cached) → SUB 2 (Filtered) per line MATCH ▼
SUB 1 (Cached): Try matching each invoice against all cached invoice rows. Steps 0→1a→1a-sub→1b→2→3.

SUB 2 (Filtered): Only triggered if SUB 1 fails. Sends a single SOAP parameter at a time (sequential fallback): P_CUSTOMER_NAME → P_INVOICE_NUM → P_INVOICE_AMOUNT → P_INVOICE_DATE until one returns data. Run same matching.
Return MatchedRecord JSON OUTPUT ▼
Original fields + fusion_* matched fields + match scenario + match reason + no_match_reason. HTTP 200.

Call Oracle BIP via SOAP ORACLE ▼
Builds SOAP 1.2 envelope → HTTP POST with Basic Auth → Parse XML response → Extract reportBytes (base64) → Decode to CSV bytes → Store in cache
SOAP Call Details client.py ▼
  • Builds XML envelope with reportAbsolutePath and optional parameterNameValues
  • Both reportAbsolutePath and parameter values escaped via xml.sax.saxutils.escape() (XML injection protection)
  • HTTP POST to ORACLE_BIP_URL with Basic Auth
  • Timeout: 120 seconds
  • HTTP 401 → AuthError → API returns 401
  • HTTP 500 / SOAP Fault → ReportError → API returns 502
  • Success → extract pub:reportBytes → base64 decode → CSV bytes
CSV Parsing & Column Normalization reports.py ▼
  • Decode bytes as utf-8-sig (handles BOM)
  • Parse with csv.DictReader
  • Normalize column names: strip() → UPPER() → spaces → _
  • Example: "Bill Customer Name" → "BILL_CUSTOMER_NAME"
  • Store in cache with current timestamp

Ambiguity rule: When both numbers ≤ 12 (e.g. 01/02/2025), defaults to DD-MM (Feb 1, not Jan 2).
Amount Matching — _amounts_match() AMOUNTS ▼
abs(abs(oracle_value) - abs(input_value)) < 0.005

  • Commas stripped from Oracle values: "1,000.50" → 1000.50
  • Absolute values compared — credit memos (negative in Oracle) still match
  • Tolerance: 0.005 (less than 1 cent) to handle floating point rounding
Substring Matching STRINGS ▼
Used in receipt A1-A3 and invoice 1a-sub, 3.

Checks if the input string is contained within the Oracle value:
"25908454" in "126125908454" → True

Handles Oracle's prefixed transaction numbers where AI extracts only the suffix.
Customer Matching CUSTOMER ▼
  • Case-insensitive: both sides .strip().lower()
  • Exact match only — not substring
  • If customer_name is null/empty in request → customer filter is skipped (not enforced)
  • Steps A4, A5, B2, B3 are skipped entirely if customer is null
  • Steps A6, A7, B4 do not require customer — ensure matches even when customer is null
Cache Logic cache.py ▼
  • In-memory Python dict: _cache = {}
  • Thread-safe: all access guarded by threading.Lock
  • Keys: "receipt", "invoice"
  • Value: {"rows": [...], "timestamp": float}
  • TTL: 300 seconds (5 minutes) — configurable via CACHE_TTL_SECONDS
  • Cache HIT: age < TTL → return rows
  • Cache STALE: age ≥ TTL → delete entry, fetch fresh
  • Cache MISS: no entry → fetch fresh, store with timestamp
  • SUB 2 calls (filtered) are never cached
